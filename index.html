Eclipse-egit-github(git9)

在eclipse环境下安装插件可以很轻松的使用github(git9),把复杂的命令行操作转化为无脑的点击右键提交@_@

好吧废话不多说，我们马上开始吧

安装和配置

  • 首先是在eclipse里安装插件egit,地址是http://download.eclipse.org/egit/updates
    • 具体的做法是选 Help->Install New Software->弹出窗口右上 add ->弹出窗口 Name: egit(随意） Location: http://download.eclipse.org/egit/updates
      写好后大概是这个样子：
    • 点ok后它会pend和analyse一段时间，会要等一下，之后把弹出来的Eclipse Git Teamwork Provider 和 JGit 都勾上，无脑next就可以了。
    • 有时候会出现有些包已经下载过或者不兼容的情况，最简单的方法就是把出错的包去掉，不过可能会丢失某些功能，所以想折腾的话还是google一下吧。
  • 安装好后，打开window->perference 在左上搜ssh2，找到SSH2点开进行配置。
    • General 中的SSH2 home 指定了存放钥匙的位置，如果是不想折腾的话就存默认的C盘吧，反正不占空间
    • 在Key Management 中点Generate RSA Key，你一直点它会不断的生成不同的，玩够了之后就点Save Private Key，然后生成的秘钥就会自动的保存到General指定的那个位置了。
      (一定要记得保存啊，然后去上面说到那个C盘下地址看看有没有文件)，
      (这里注意，用git bash 命令行生成的秘钥Comment用的是用户的邮箱，这里是RSA-1024，这个问题暂时不懂)
    • 之后把得到的秘钥复制下来，在你的github(git9)账户上登记一下这个秘钥。
      登记的地方是用户下的Edit Your Profile 里的SSH Key

    • 点击new key，title你记得就好，key是把刚才复制的秘钥粘进去。

    • 在eclipse 里建一个helloworld 项目，随便写点东西，只要有文件就行。
    • 【偷个懒。。。等会再写】

克隆项目

  • 从网上克隆一个项目到本地
    • 首先打开Git Respositories 视角，就是Window->Show View->Git Respositories 然后这个页面就会出现在Eclipse的某个角落
    • 点右上一群黄色按钮中有一个叫做 Clone a Git Respositories… 的东西，在弹出的对话框内，URL写你项目的URL
      这个URL一般在项目管理页面可以找到。URL复制进去后，Host 和 Respository path会自动补全，Protocol根据你的URL选择对应的。
      由于http/https 协议需要填用户名和密码，所以这里我们用SSH协议（这里选择后会给URL自动补全）
    • next之后系统会连接，这里连接可能会出现各种问题，很大概率是秘钥没有配好导致的。
      有错误可以自行google。成功的画面是这样的：
    • 然后就是选一个地方存放代码，Remote name是在多个地方更改项目作为区分用的，暂时就用默认名字吧。
    • 项目下载下来之后，可能还没有配项目文件什么的，所以无法在Eclipse导入，所以就自己新建一个Tomcat 工程，然后把.project之类的配置文件全部拷过去，然后改一下文件里的项目名字就好了。
      打入后是这样的，凡是和网上关联的文件夹/文件都会多个黄色的棒子，没有关联的会有问号，可以右键 Team->add to Index 添加关联
    • 测试克隆是否成功，在项目里随便找一个文件，比如这里的README.md，在里面加几行代码，
      然后在Package Explorer 工程文件夹上点右键->Team->Commit
      Commit message 是必填的，最好注明版本号和修改内容，这个由各个开发团队而定。
      然后选中你要提交更改的文件，点Commit and Push就好了。如果只是Commit的话并没有提交到网上，只是在本地保存了修改，还要手动push一下。
    • 之后可以去网上项目管理页面查看是否提交成功。
